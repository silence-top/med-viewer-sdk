# 主题系统

MedViewer SDK 内置基于 **CSS Custom Properties（CSS 变量）** 的主题引擎，支持预设主题切换、运行时动态切换及完全自定义。

## 预设主题

| 名称 | 说明 | 强调色 |
|------|------|--------|
| `'default'` | 默认暗色主题（深蓝灰底，绿松石强调） | `#31d0aa` |
| `'light'` | 白天主题（白色底，蓝色强调） | `#1890ff` |
| `'dark'` | 深黑主题（近黑底，紫色强调） | `#bb86fc` |

## 主题配置方式

### 1. 初始化时指定

```ts
// 方式一：预设名称字符串
new MedViewerEngine({
  osdOptions: { id: 'viewer', tileSources: '...' },
  theme: 'light'   // 'default' | 'light' | 'dark'
})

// 方式二：局部覆盖（基于 default 主题合并）
new MedViewerEngine({
  osdOptions: { id: 'viewer', tileSources: '...' },
  theme: {
    accentColor: '#7c3aed',
    accentHover: '#6d28d9',
    iconFilter: 'none'
  }
})

// 方式三：完整命名主题
import { PRESET_THEMES, type ThemeDefinition } from 'med-viewer-sdk'

const myTheme: ThemeDefinition = {
  name: 'brand',
  colors: {
    ...PRESET_THEMES.light.colors,  // 继承 light
    accentColor: '#f97316',         // 覆盖强调色
    accentHover: '#ea580c'
  }
}

new MedViewerEngine({
  osdOptions: { id: 'viewer', tileSources: '...' },
  theme: myTheme
})
```

### 2. 不传 theme（使用默认主题）

```ts
new MedViewerEngine({ osdOptions: { ... } })
// 等同于 theme: 'default'
```

## 运行时切换

```ts
// 切换预设主题
engine.setTheme('dark')

// 局部覆盖当前主题颜色
engine.setTheme({ panelBackground: 'rgba(255,0,0,0.8)' })

// 获取当前主题名称
engine.getThemeName()   // 'default' | 'light' | 'dark' | 'custom' | 自定义name

// 获取当前全部颜色值
const colors = engine.getThemeColors()
console.log(colors?.accentColor)
```

## Vue 动态绑定

Vue 适配器提供 `theme` prop，支持响应式绑定：

```vue
<template>
  <MedViewer :options="options" :theme="currentTheme" />
  <button @click="currentTheme = 'light'">白天</button>
  <button @click="currentTheme = 'dark'">深黑</button>
  <button @click="currentTheme = 'default'">默认</button>
</template>

<script setup>
import { ref } from 'vue'
import { MedViewer } from 'med-viewer-sdk'

const currentTheme = ref('default')
const options = { osdOptions: { id: 'viewer', tileSources: '...' } }
</script>
```

`theme` prop 变化时，SDK 内部会自动调用 `engine.setTheme()`，无需手动操作。

## 自定义主题最佳实践

### 继承预设 + 局部覆盖（推荐）

```ts
import { PRESET_THEMES } from 'med-viewer-sdk'

const brandTheme = {
  name: 'brand',
  colors: {
    ...PRESET_THEMES.light.colors,
    accentColor: '#f97316',
    accentHover: '#ea580c',
    accentBgLow: 'rgba(249, 115, 22, 0.08)',
    accentBgHigh: 'rgba(249, 115, 22, 0.15)',
    checkboxCheckedBg: '#f97316',
    sliderThumbColor: '#f97316',
    annoInnerStroke: '#f97316',
    annoEditorBtnBg: '#f97316',
    annoEditorBtnHoverBg: '#ea580c',
    selectionColor: '#f97316',
    selectionHandleBorder: '#f97316',
    iconFilter: 'invert(1)'   // light 主题需要 invert
  }
}

engine.setTheme(brandTheme)
```

### 仅改几个颜色

```ts
// Partial<ThemeColors> 方式，未指定的字段继承 default 主题
engine.setTheme({
  accentColor: '#e11d48',
  accentHover: '#be123c'
})
```

## PNG 图标主题适配

工具栏图标（`tool_anno` / `tool_selection` / `tool_color` / `tool_reset` / `tool_rotate`）为白色 PNG，在深色背景下可见。

切换到浅色主题时，通过 `iconFilter: 'invert(1)'` 将白色图标反转为深色，在浅色背景下保持可读性。

| 主题 | `iconFilter` 值 | 效果 |
|------|----------------|------|
| `default` | `'none'` | 白色图标，适配深蓝灰底 |
| `light` | `'invert(1)'` | 图标反转为深色，适配白底 |
| `dark` | `'none'` | 白色图标，适配纯黑底 |

**自定义主题指导原则**：
- 深色背景 → `iconFilter: 'none'`
- 浅色背景 → `iconFilter: 'invert(1)'` 或使用精确的 `invert(x) sepia(x) ...` 滤镜

## 注意事项与边界

1. **CSS 变量作用域**：变量注入到 OSD Viewer 容器元素，不会污染宿主页面全局样式。
2. **Partial 合并基准**：传入 `Partial<ThemeColors>` 时，始终以 `default` 主题为基准合并，而非当前主题。如需基于当前主题合并，请先读取 `engine.getThemeColors()` 后手动合并。
3. **SelectionPlugin / ScalebarPlugin** 部分动态颜色通过 `updateTheme()` 内部方法处理，调用 `engine.setTheme()` 时会自动触发，无需手动调用。
4. **annotorious 编辑器样式**：通过 CSS 优先级覆盖 `annotorious.min.css` 默认样式，注入在组件内部。主题切换时，已注入的 `<style>` 标签会被替换。
5. **未知预设名**：传入不存在的预设名称时，会打印警告并回退到 `default` 主题。
