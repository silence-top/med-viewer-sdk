## 项目定位

MedViewer SDK 是一个面向医学影像分析场景的前端 SDK，封装了：

- **OpenSeadragon**（OSD）作为底层深度缩放引擎
- **Annotorious v2.7.17**（fork 自 `annotorious-openseadragon-ld`）作为矢量标注引擎
- 自研的工具栏、颜色调整、区域截图、比例尺、倍率显示等插件

SDK 以 **UMD + ESM 双格式**发布，兼容原生 HTML 引入和 Vue 2.7/3 组件化集成，无 React 适配器（可后续按需添加）。

## 核心模块说明

### Engine.ts — 统一入口

**核心职责**：初始化 OSD、按配置顺序初始化各插件、暴露公共 API。

**关键初始化顺序（不可随意调换）**：
1. `OpenSeadragon()` 创建 Viewer
2. `new ThemeManager()` — 必须在所有插件之前，确保 CSS 变量在插件注入样式前已就绪
3. `ScalebarPlugin` — 先于 Annotorious，避免 DOM 冲突
4. `AnnoAnnotator` — **必须等 OSD `open` 事件后才能初始化**（内部已通过 `addOnceHandler('open', ...)` 处理），否则会抛出 "Cannot set properties of undefined (setting 'display')" 错误
5. `MedToolbar` / `SelectionPlugin` / `ColorAdjustPlugin` / `MagnificationPlugin`

**事件系统**：Engine 实现了一个轻量事件总线（`addHandler`/`addOnceHandler`/`removeHandler`/`emit`），目前仅暴露 `ready` 事件（OSD `open` 后触发）。

---

### Theme.ts — 主题系统

**设计思路**：所有样式通过 **CSS Custom Properties** 注入到 OSD 容器元素，各插件的 `injectStyles()` 中用 `cssVar('xxx')` 引用变量，做到切换主题时只需替换变量值。

**主题切换流程**：
```
engine.setTheme(config)
  → ThemeManager.setTheme()       // 更新 CSS 变量到容器元素
  → SelectionPlugin.updateTheme() // Selection 的边框/手柄颜色是 JS 对象，需手动更新
  → ScalebarPlugin.updateTheme()  // 同上
```

**注意**：`injectStyles()` 中的 `<style>` 标签是一次性注入到 `document.head` 的，CSS 变量引用会在变量变化时自动生效，不需要重新注入。但 SelectionPlugin 的边框样式是通过 JS 对象配置传给 OSD Selection 插件的，不走 CSS 变量，所以需要 `updateTheme()` 手动同步。

**45 个颜色变量**的完整定义和默认值见 `Theme.ts` 中的 `ThemeColors` 接口。

---

### Toolbar.ts — 工具栏

工具栏通过纯 DOM 操作构建，**不依赖任何 UI 框架**。

**按钮系统**：
- `ToolbarButton` 接口支持 `onClick`（点击回调）、`dropdownContent`（下拉面板）、`activeEvent`（外部事件驱动的激活态）
- 预设按钮（标注、颜色调整、截图）在 Toolbar 内部硬编码，通过 `engine.anno`/`engine.colorAdjust`/`engine.selection` 引用
- 自定义 `buttons` 数组会完全**覆盖**预设按钮（不是追加）

**工具栏图标**：`src/assets/icons/` 下的 PNG 均为白色图标（适配深色背景）。切换浅色主题通过 `iconFilter: 'invert(1)'` CSS filter 反转，不修改图标资源本身。

---

### AnnoAnnotator.ts — 标注

封装 `annotorious-openseadragon-ld`（基于 Annotorious v2.7.17 的定制 fork）。

**两层事件**：
1. Annotorious 原生事件（`createAnnotation`、`updateAnnotation` 等）→ 通过 `config.onXxx` 回调透传给业务层
2. SDK 内部 `modeChange` 事件 → 供工具栏按钮感知绘制状态变化

**测量标签**：通过 `ShapeLabelsFormatter.js` 实现，注入为 Annotorious 的 `formatter` 选项。计算逻辑在 `plugins/ShapeLabelsFormatter.js`，依赖 `svg-path-properties` 库解析 SVG path 数据（主要用于 freehand 曲线周长/面积计算）。

**Annotorious 样式覆盖**：在 `injectStyles()` 中通过高优先级 CSS 规则覆盖 `annotorious.min.css` 默认样式，所有颜色引用 CSS 变量。

---

### ColorAdjustPlugin.ts — 颜色调整

**性能优化点**：
1. **LUT（查找表）**：将亮度、对比度、反色、伽马预计算为 256 长度数组，避免像素循环内部重复运算
2. **异步调度**：像素处理推入 `setTimeout(..., 0)` 宏任务队列，让出主线程给拖拽/缩放
3. **防抖**：`setAdjustments()` 内置 `debounceMs`（默认 60ms），避免滑块拖拽造成大量重绘请求
4. **执行路径分支**：greyscale → sepia → hue/saturation 按有无启用分支执行，避免无效计算

依赖 `openseadragon-filtering` 插件提供的 `viewer.setFilterOptions()` 接口。

---

### SelectionPlugin.ts — 区域截图

**截图 Blob 生成原理**：
1. OSD Selection 插件回调返回视口坐标系的 `rect`
2. 通过 `viewport.imageToViewportRectangle` + `viewport.viewportToViewerElementRectangle` 转换为屏幕坐标
3. 从 `viewer.drawer.canvas` 上 `drawImage` 截取对应区域到新 Canvas
4. `toBlob()` 转换为 Blob 回调业务层

**CORS 限制**：截图依赖浏览器允许读取跨域 Canvas 内容，图像服务必须配置 CORS，且 OSD 需 `crossOriginPolicy: 'Anonymous'`。

---

### Magnification.ts — 倍率

**两套倍率计算逻辑**，由 `type` 选项控制：

- **OSD 模式**（标准）：`currentViewportZoom → imageZoom → baseMag (from metadata.max_magnification) → 显示倍率`
- **LD 模式**（私有协议）：
  ```
  conversionFactor = (20 × 0.0011 × pixelsPerMeter) / imageWidth
  magnification = conversionFactor × currentViewportZoom
  ```
  这个公式是联影私有协议的经验公式，不适用于其他切片系统。

## 关键设计决策记录

### 1. 图标使用 PNG + CSS filter 而非 SVG

- 现状：工具栏图标为白色 PNG
- 主题适配：通过 `iconFilter` CSS 变量 + `filter: invert(1)` 实现浅色主题适配
- 原因：业务方可能使用自定义图标（任意格式），统一改 SVG 会造成接入成本；CSS filter 方案对图标格式无要求
- **注意**：如果后续图标非单色（如彩色图标），`invert(1)` 会产生颜色偏差，届时需要改用 SVG + currentColor 或提供单独的主题色版本图标

### 2. CSS-in-JS 注入到 document.head（而非 Shadow DOM）

- 各插件的 `injectStyles()` 将 `<style>` 标签注入 `document.head`，通过 `styleId` 检查去重
- 优点：简单，无需 Shadow DOM 兼容处理
- 缺点：样式是全局的，在同一页面中存在多个 Viewer 实例时，**样式不隔离**
- 已知风险：Annotorious 编辑器样式（`.r6o-editor` 等）注入一次后不会随主题切换而更新，因为 `styleId` 检查跳过了重注入；目前通过 CSS 变量本身实现动态更新（变量值变了，样式自动生效），逻辑正确但易产生误解

### 3. Annotorious 使用 fork 版本

- 包名：`annotorious-openseadragon-ld`（非官方 `@recogito/annotorious-openseadragon`）
- 这是一个内部定制版本，包含了特定业务需求的修改
- **升级风险极高**：Annotorious v3 架构完全重写，API 不兼容；如需升级，所有标注相关代码需全量重写

### 4. vue-demi 实现 Vue 2/3 通用组件

- `MedViewer.ts` 使用 `vue-demi` 实现一套代码兼容 Vue 2.7+ 和 Vue 3
- vue-demi 在编译时根据 vue 版本自动适配
- 注意：Vue 2.7 之前的版本（如 2.6）不支持 Composition API，不在支持范围内

### 5. Annotorious 初始化时序问题

- Annotorious 必须在 OSD Canvas 元素创建完成后才能初始化
- OSD Canvas 在 `open` 事件触发时已就绪
- Engine.ts 通过 `viewer.addOnceHandler('open', () => mountAnnotorious())` 解决此问题
- 如果 tileSources 是同步加载的本地对象，OSD 可能在构造时就已 `open`，Engine 内有 `viewer.isOpen()` 的分支判断处理此情况

## 构建与发布流程

### 本地开发

```bash
# 安装依赖
npm install

# 直接打开 index.html 进行调试（无 dev server，直接 ESM 引入）
# 确保 index.html 中 tileSources 指向可访问的图像服务
```

> 项目没有配置 `vite dev` 服务器，`index.html` 是直接用 ES Module 从 `src/index.ts` 引入的，依赖浏览器原生 ESM 支持。开发时直接用浏览器打开或配置一个简单的静态服务器。

### 构建

```bash
npm run build
```

构建命令实际执行三步：
1. `vite build` — 生成 `dist/med-viewer-sdk.mjs`（ESM）和 `dist/med-viewer-sdk.umd.js`（UMD）
2. `tsc -p tsconfig.dts.json` — 生成类型声明，输出到 `dist/index.d.ts`
3. PowerShell 脚本 — 将 `dist/index.d.ts` 重命名为 `dist/med-viewer-sdk.d.ts`，删除中间产物目录

**构建产物**：

```
dist/
├── med-viewer-sdk.mjs      # ESM
├── med-viewer-sdk.umd.js   # UMD（含 window.MedViewerSDK 全局变量）
├── med-viewer-sdk.d.ts     # TypeScript 类型声明
└── style.css               # 样式（openseadragon + annotorious + SDK 自身）
```

### 版本管理

当前版本：`0.1.25`（`package.json` 的 `version` 字段）。发布前手动更新版本号，没有自动化脚本。

## 已知问题与技术债

### 高优先级

1. **Annotorious 测量标签硬编码 µm 单位**
   - 位置：`ShapeLabelsFormatter.js` → `getWithUnit()` 函数
   - 问题：单位被注释掉了，目前统一输出 `μm`，没有根据尺寸自动换算（µm / mm / m）
   - 影响：大尺寸测量结果显示异常（如显示 1000000μm 而非 1m）

2. **Annotorious 编辑器样式非主题化的问题**
   - 位置：`AnnoAnnotator.ts → injectStyles()`
   - 问题：`styleId` 去重逻辑导致样式 `<style>` 标签只注入一次；虽然 CSS 变量本身是响应式的，但如果页面中有多个 Viewer 实例，只有第一个实例的样式生效（共享同一个 `styleId`）
   - 影响：多 Viewer 实例场景下 Annotorious 主题可能不正确

3. **SelectionPlugin 截图坐标在旋转图像时可能不准确**
   - OSD viewport 坐标转换在图像有旋转角度时存在偏差
   - 尚未验证旋转角度下的截图准确性

### 中优先级

4. **工具栏预设按钮与插件的强耦合**
   - `Toolbar.ts` 中的预设按钮直接访问 `engine.anno`/`engine.colorAdjust`/`engine.selection`
   - 当相关插件未启用时，按钮会静默失效（无错误提示）
   - 如果插件顺序或命名变更，工具栏预设按钮会失效

5. **LD 倍率换算公式缺乏文档**
   - `Magnification.ts` 中的 `conversionFactor = (20 × 0.0011 × pixelsPerMeter) / width` 公式来源不明
   - 没有单元测试，换算结果的正确性依赖实际设备验证

6. **CSS 变量作用域限于 OSD 容器元素，但 Annotorious 编辑器渲染在 document.body 下**
   - `.r6o-editor` 是挂载在 `document.body` 的绝对定位元素，不在 OSD 容器的 CSS 作用域内
   - 当前通过在 `document.head` 注入全局样式规则解决，但这破坏了"样式作用域限于 Viewer 容器"的设计目标

### 低优先级

7. **没有单元测试**
   - 整个项目没有任何测试（`package.json` 中 `test` 命令为空）
   - 核心逻辑（坐标换算、LUT 计算、主题合并）建议补充单元测试

8. **ShapeLabelsFormatter.js 未完成 TypeScript 迁移**
   - 仍为 `.js` 文件，存在 `type.d.ts` 中的手写类型声明
   - 与项目其余 `.ts` 文件风格不一致

## 扩展指南

### 添加新插件

1. 在 `src/core/` 创建新插件类（参考 `Scalebar.ts` 或 `Magnification.ts`）
2. 在 `MedEngineOptions.plugins` 中添加配置字段（`Engine.ts`）
3. 在 `Engine.ts → initPlugins()` 中按顺序初始化
4. 在 `Engine.ts` 类上添加公开属性（`public myPlugin: MyPlugin | null = null`）
5. 在 `Engine.ts → destroy()` 中调用 `this.myPlugin?.destroy()`
6. 在 `src/index.ts` 中导出新插件类和相关类型

如插件需要感知主题变化：
- 实现 `updateTheme(colors: ThemeColors): void` 方法
- 在 `Engine.ts → setTheme()` 中调用

### 添加新标注器

继承 `BaseAnnotator` 抽象类，实现全部抽象方法：

```ts
export class MyAnnotator extends BaseAnnotator {
  setEnabled(enabled: boolean): void { ... }
  setTool(tool: string | null): void { ... }
  getAnnotations(): any[] { ... }
  setAnnotations(data: any[]): void { ... }
  clear(): void { ... }
  destroy(): void { ... }
}
```

### 添加新主题颜色变量

1. 在 `ThemeColors` 接口中添加新字段（`Theme.ts`）
2. 在 `DEFAULT_COLORS`、`LIGHT_COLORS`、`DARK_COLORS` 中各自添加对应值
3. 在 `CSS_VAR_MAP` 中添加变量名映射（命名约定：`--med-xxx`）
4. 在需要引用的插件中使用 `cssVar('newKey')`
