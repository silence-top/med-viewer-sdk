## 原生 HTML（无框架）

通过 UMD 包直接引入，挂载到 `window.MedViewerSDK`。

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <style>
    #viewer { width: 100%; height: 100vh; }
  </style>
</head>
<body>
  <div id="viewer"></div>

  <!-- 1. 先引入 peer dependencies -->
  <script src="https://cdn.jsdelivr.net/npm/openseadragon@3/build/openseadragon/openseadragon.min.js"></script>

  <!-- 2. 引入 SDK -->
  <link rel="stylesheet" href="./dist/style.css">
  <script src="./dist/med-viewer-sdk.umd.js"></script>

  <script>
    const { MedViewerEngine } = window.MedViewerSDK

    const engine = new MedViewerEngine({
      osdOptions: {
        id: 'viewer',
        tileSources: 'https://your-server/iiif/image/info.json'
      },
      locale: 'zh-CN',
      theme: 'default',
      plugins: {
        toolbar: true,
        colorAdjust: true,
        selection: {
          onSelection: (rect, blob) => {
            console.log('截图区域:', rect)
          }
        },
        scalebar: { type: 1, pixelsPerMeter: 250000 },
        magnification: true,
        annotorious: {
          onCreateAnnotation: (anno) => {
            // 保存到服务器
            fetch('/api/annotations', {
              method: 'POST',
              body: JSON.stringify(anno)
            })
          }
        }
      }
    })

    engine.addHandler('ready', () => {
      console.log('图像加载完成')
    })
  </script>
</body>
</html>
```

## Vue 3 集成

### 方式一：使用 `<MedViewer>` 组件（推荐）

```vue
<template>
  <div class="viewer-container">
    <MedViewer
      ref="viewerRef"
      :options="viewerOptions"
      :theme="currentTheme"
      @ready="onReady"
    />
    <div class="controls">
      <button @click="currentTheme = 'light'">白天</button>
      <button @click="currentTheme = 'default'">默认</button>
      <button @click="currentTheme = 'dark'">深黑</button>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, onBeforeUnmount } from 'vue'
import { MedViewer, type MedEngineOptions, type MedViewerEngine } from 'med-viewer-sdk'

const currentTheme = ref<'default' | 'light' | 'dark'>('default')
const viewerRef = ref()
let engine: MedViewerEngine | null = null

const viewerOptions: MedEngineOptions = {
  osdOptions: {
    id: 'viewer',
    tileSources: 'https://your-server/iiif/image/info.json'
  },
  locale: 'zh-CN',
  plugins: {
    toolbar: true,
    colorAdjust: true,
    scalebar: { pixelsPerMeter: 250000 },
    magnification: { type: 'LD', pixelsPerMeter: 250000 },
    annotorious: {
      onCreateAnnotation: (anno) => emit('annotation-create', anno)
    }
  }
}

const onReady = (eng: MedViewerEngine) => {
  engine = eng
  // 直接操作引擎
  engine.addHandler('ready', () => {
    console.log('图像加载完成')
  })
}
</script>

<style scoped>
.viewer-container {
  position: relative;
  width: 100%;
  height: 100vh;
}
</style>
```

### 访问 engine 实例

```ts
// 通过 ref 暴露的 engine
const viewerRef = ref()
// 组件挂载后访问
viewerRef.value?.engine  // ShallowRef<MedViewerEngine | null>

// 或通过 @ready 回调获取
const onReady = (eng) => {
  engine = eng  // 直接操作引擎
}
```

### 方式二：直接使用 MedViewerEngine

适用于需要更精细控制的场景：

```vue
<template>
  <div ref="containerEl" class="viewer"></div>
</template>

<script setup lang="ts">
import { ref, onMounted, onBeforeUnmount } from 'vue'
import { MedViewerEngine } from 'med-viewer-sdk'

const containerEl = ref<HTMLElement | null>(null)
let engine: MedViewerEngine | null = null

onMounted(() => {
  engine = new MedViewerEngine({
    osdOptions: {
      element: containerEl.value!,  // 传 element 而非 id
      tileSources: '...'
    },
    plugins: { toolbar: true }
  })
})

onBeforeUnmount(() => {
  engine?.destroy()
  engine = null
})
</script>
```

## Vue 2.7 集成

SDK 通过 `vue-demi` 同时支持 Vue 2.7+ 和 Vue 3。

```vue
<template>
  <MedViewer :options="options" theme="default" @ready="onReady" />
</template>

<script>
import { MedViewer } from 'med-viewer-sdk'

export default {
  components: { MedViewer },
  data() {
    return {
      options: {
        osdOptions: { id: 'viewer', tileSources: '...' },
        plugins: { toolbar: true }
      }
    }
  },
  methods: {
    onReady(engine) {
      this.engine = engine
    }
  },
  beforeDestroy() {
    this.engine?.destroy()
  }
}
</script>
```

## 完整功能示例

```ts
import {
  MedViewerEngine,
  ScalebarType,
  ScalebarLocation,
  MagnificationType,
  MagnificationPosition,
  ToolbarPosition,
  PRESET_THEMES
} from 'med-viewer-sdk'

const engine = new MedViewerEngine({
  osdOptions: {
    id: 'viewer',
    tileSources: 'https://your-server/image.dzi',
    navigatorPosition: 'TOP_RIGHT'
  },
  locale: 'zh-CN',
  theme: {
    ...PRESET_THEMES.light.colors,
    accentColor: '#7c3aed'  // 局部品牌色覆盖
  },
  plugins: {
    toolbar: {
      position: ToolbarPosition.MIDDLE_LEFT
    },
    colorAdjust: {
      adjustments: { brightness: 1.1 },
      debounceMs: 60
    },
    selection: {
      allowRotation: true,
      restrictToImage: true,
      onSelection: async (rect, blob) => {
        if (blob) {
          const formData = new FormData()
          formData.append('file', blob, 'crop.png')
          await fetch('/api/upload', { method: 'POST', body: formData })
        }
      }
    },
    scalebar: {
      type: ScalebarType.MICROSCOPY,
      pixelsPerMeter: 250000,
      location: ScalebarLocation.BOTTOM_LEFT
    },
    magnification: {
      type: MagnificationType.OSD,
      position: MagnificationPosition.MIDDLE_LEFT
    },
    annotorious: {
      onCreateAnnotation: (anno) => console.log('new', anno),
      onUpdateAnnotation: (anno, prev) => console.log('update', anno),
      onDeleteAnnotation: (anno) => console.log('delete', anno)
    }
  }
})

engine.addHandler('ready', () => {
  // 加载历史标注数据
  engine.anno?.setAnnotations(savedAnnotations)
  // 加载 AI 检测框
  engine.loadAIMarks(aiResults, true)
})
```

## Vite 项目配置

```ts
// vite.config.ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  optimizeDeps: {
    // 如遇到 openseadragon 预打包问题，排除它
    exclude: ['openseadragon']
  }
})
```

如需将 SDK 作为外部依赖（不打包进业务代码）：

```ts
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      external: ['med-viewer-sdk', 'openseadragon', 'vue'],
      output: {
        globals: {
          'med-viewer-sdk': 'MedViewerSDK',
          openseadragon: 'OpenSeadragon',
          vue: 'Vue'
        }
      }
    }
  }
})
```

## 标注数据持久化

SDK 不内置持久化逻辑，业务层需自行实现。

### 保存

```ts
plugins: {
  annotorious: {
    onCreateAnnotation: async (anno) => {
      await fetch('/api/annotations', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(anno)
      })
    },
    onUpdateAnnotation: async (anno, prev) => {
      await fetch(`/api/annotations/${anno.id}`, {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(anno)
      })
    },
    onDeleteAnnotation: async (anno) => {
      await fetch(`/api/annotations/${anno.id}`, { method: 'DELETE' })
    }
  }
}
```

### 加载

```ts
engine.addHandler('ready', async () => {
  const response = await fetch(`/api/annotations?imageId=${imageId}`)
  const annotations = await response.json()
  engine.anno?.setAnnotations(annotations)
})
```

## 常见问题

### Q: 图像显示空白/黑屏

1. 检查 `osdOptions.id` 对应的 DOM 元素是否已存在
2. 检查图像服务 CORS 配置（需允许跨域）
3. 检查 `tileSources` 路径是否正确，浏览器 Network 面板确认请求状态

### Q: 标注工具无法点击

确认 Annotorious 是在 OSD `open` 事件之后初始化的——SDK 内部已处理，但若手动管理生命周期，需在 `engine.addHandler('ready', ...)` 后操作 `engine.anno`。

### Q: 截图 blob 为 null

图像服务需在响应头中包含：
```
Access-Control-Allow-Origin: *
```
且 OSD 配置需设置 `crossOriginPolicy: 'Anonymous'`（SDK 默认已配置）。

### Q: 颜色调整不生效

1. 检查浏览器是否支持 WebGL（`chrome://gpu/` 查看状态）
2. 确保图像已完全加载（在 `ready` 事件后调用 `setAdjustments`）

### Q: Vue 组件销毁后报错

确保在组件卸载时（`onBeforeUnmount`）调用 `engine.destroy()`。使用 `<MedViewer>` 组件时 SDK 已自动处理。

### Q: 主题切换后图标颜色没有变化

工具栏图标为白色 PNG，切换到浅色主题需将 `iconFilter` 设置为 `'invert(1)'`。使用预设 `light` 主题时已自动配置；自定义主题需手动设置。

### Q: 倍率显示为 "-"

OSD 模式下图像源需包含 `max_magnification` 字段。若使用 LD 协议，请将 `type` 设置为 `MagnificationType.LD` 并配置 `pixelsPerMeter`。

### Q: 如何在多个 Viewer 实例间共享标注数据

每个 `MedViewerEngine` 实例独立管理自己的 `anno`，通过各自的回调同步到后端，再通过 `setAnnotations()` 加载到另一个实例。
